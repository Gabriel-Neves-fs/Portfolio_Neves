# Visao do projeto

## Objetivo

Construir um portfolio pessoal moderno, responsivo e facil de manter para apresentar Gabriel Neves como profissional em transicao consistente de Analista de Suporte Tecnico Pleno para Desenvolvedor Full Stack.

Repositorio oficial: `https://github.com/Gabriel-Neves-fs/Portfolio_Neves`

## Narrativa principal

O portfolio deve mostrar que a experiencia em suporte tecnico nao e um desvio da carreira de desenvolvimento. Ela e uma vantagem competitiva.

A narrativa central:

- Experiencia real com sistemas em producao.
- Investigacao de incidentes, leitura de logs, APIs e integracoes.
- Comunicacao com times internos e fornecedores.
- Evolucao para construcao de produtos com Next.js, TypeScript, Node.js e PostgreSQL.

## Identidade visual

Direcao atual:

- Editorial tecnico.
- Profissional, direto e com personalidade.
- Layout com secoes bem marcadas.
- Tipografia forte no hero.
- Cards objetivos para stack, projetos e experiencia.

Evitar:

- Visual generico de template.
- Excesso de gradientes decorativos.
- Textos longos sem hierarquia.
- Copiar estrutura visual de referencias externas literalmente.

## Publico

O site deve funcionar para:

- Recrutadores tecnicos.
- Tech leads e pessoas desenvolvedoras avaliando repertorio.
- Contatos profissionais vindos de LinkedIn, GitHub ou indicacoes.

## Decisao tecnica inicial

O projeto nao tera backend no inicio.

Motivo:

- O conteudo muda pouco.
- Arquivos TypeScript locais facilitam manutencao.
- Deploy na Vercel fica simples.
- Reduz complexidade enquanto o foco e apresentacao profissional.
