# Documentacao interna do portfolio

Esta pasta guarda anotacoes de projeto que ajudam a manter o portfolio evoluindo com criterio, sem misturar documentacao interna com a interface publica.

Por estar em uma pasta iniciada com ponto (`.docs`), ela fica visualmente oculta em muitos exploradores de arquivo, mas ainda pode ser versionada no Git caso voce queira manter esse historico no repositorio.

## Arquivos

- `project-vision.md`: objetivo, identidade e narrativa profissional do portfolio.
- `backlog.md`: proximas tasks organizadas por objetivo, criterio de conclusao e resultado esperado.
- `content-guide.md`: como atualizar projetos, skills, experiencia e contatos sem quebrar a estrutura.

## Regra de manutencao

Sempre que uma secao nova for adicionada ao site, atualize o backlog e o guia de conteudo se isso mudar a forma de manter o projeto.
